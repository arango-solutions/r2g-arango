# Security Policy

## Supported versions

`r2g` is an experimental reference implementation. Only the `main`
branch receives security fixes; point releases on PyPI are supported
until the next release.

| Version          | Supported |
| ---------------- | --------- |
| `main` (HEAD)    | yes       |
| latest PyPI      | yes       |
| older PyPI       | no        |

## Reporting a vulnerability

Please do **not** open a public issue for security vulnerabilities.

Report privately by opening a
[GitHub Security Advisory](https://github.com/ArthurKeen/r2g-arango/security/advisories/new)
on this repository. If you cannot use GitHub advisories, email
`aakeen@yahoo.com` with the subject line `r2g security:` followed by
a short description.

Include in your report:

- A description of the issue and its impact.
- Steps to reproduce, or a proof-of-concept, if you have one.
- Affected version(s) or commit SHA.
- Any suggested mitigation or fix.

## Disclosure process

1. We will acknowledge receipt within five business days.
2. We will investigate, confirm the issue, and work on a fix.
3. Once a fix is ready and released, we will publish a security
   advisory crediting the reporter (unless you prefer anonymity) and
   describing the issue, affected versions, and mitigation.
4. For critical issues we aim to release a fix within 30 days of
   confirmation. Less severe issues may be bundled into the next
   regular release.

Thank you for helping keep `r2g` and its users safe.
